# Markdown（.md）编写格式说明书（实用版）

> 适用于：
>
> * Cursor / VS Code
> * README
> * 学习笔记
> * 技术文档
> * 项目说明

---

## 一、Markdown 是什么？

Markdown 是一种**轻量级标记语言**：

* ✅ 用**纯文本**写
* ✅ 用**符号表示结构**
* ✅ 最终由“预览器”渲染成排版好的文档

👉 **你写的是结构，不是样式**。

---

## 二、文件基本结构（推荐模板）

```md
# 文档标题

> 一句话说明这份文档是干什么的

---

## 一、章节一标题
内容内容内容

## 二、章节二标题
内容内容内容
```

---

## 三、标题（最重要的结构）

```md
# 一级标题（整篇只有一个）
## 二级标题
### 三级标题
#### 四级标题
```

规则：

* `#` 后面一定要有空格
* **不要跳级**（不要直接从 `#` 跳到 `###`）

✅ 正确示例：

```md
# 项目说明
## 功能介绍
### 登录模块
```

---

## 四、段落与换行

### 普通段落

```md
这是第一行文字。

这是第二段文字。
```

📌 **空一行 = 新段落**

---

### 强制换行（不常用）

```md
第一行  
第二行
```

（行尾两个空格）

---

## 五、强调文字

### 加粗

```md
**重要内容**
```

### 斜体

```md
*强调内容*
```

### 删除线

```md
~~已废弃内容~~
```

---

## 六、列表（非常常用）

### 无序列表

```md
- 第一项
- 第二项
  - 子项
```

### 有序列表

```md
1. 第一步
2. 第二步
3. 第三步
```

📌 列表层级靠**缩进 2 个空格**

---

## 七、代码（程序员核心）

### 行内代码

```md
使用 `Ctrl + Shift + V` 打开预览
```

### 代码块（强烈推荐）

````md
```python
def hello():
    print("Hello World")
````

````

常用语言标识：
- `python`
- `bash`
- `json`
- `js`
- `html`

---

## 八、图片（你刚学会的重点）

### 基本语法
```md
![图片说明](./images/demo.png)
````

规则：

* 使用**相对路径**
* 图片统一放在 `images/` 目录

📌 **编辑区看不到图，预览才能看到（正常）**

---

## 九、链接

### 普通链接

```md
[OpenAI 官网](https://openai.com)
```

### 引用文内标题（GitHub 支持）

```md
[跳到标题](#三、标题)
```

---

## 十、引用块（说明 / 提示）

```md
> 这是一个重要说明  
> 可以写多行
```

常用于：

* 注意事项
* 作者说明
* 教学提示

---

## 十一、分割线

```md
---
```

用于：

* 章节分隔
* 视觉停顿

---

## 十二、表格（进阶但很实用）

```md
| 名称 | 说明 | 备注 |
|------|------|------|
| Python | 后端语言 | 必学 |
| FastAPI | Web 框架 | 推荐 |
```

📌 对齐不影响渲染，只影响可读性

---

## 十三、TODO / 任务清单

```md
- [x] 已完成
- [ ] 待完成
```

非常适合：

* 学习计划
* 项目进度

---

## 十四、推荐的文件组织方式（强烈建议）

```
docs/
├─ README.md
├─ 基础说明.md
├─ 进阶说明.md
└─ images/
   ├─ img-001.png
   └─ img-002.png
```

---

## 十五、Markdown 编写最佳实践（经验总结）

✅ 一个 md 只干一件事
✅ 标题 > 列表 > 代码 > 图片
✅ 不追求花哨，追求清晰
❌ 不用绝对路径
❌ 不在 md 里堆大段无结构文字

---

## 十六、你现在最推荐的写作流程

1. 写 md（左侧）
2. `Ctrl + Shift + V` 看预览（右侧）
3. 截图 → `Ctrl + Alt + V` 插图
4. 统一 images 目录

👉 **这是“工程师级文档习惯”**
